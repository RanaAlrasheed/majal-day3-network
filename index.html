TCP vs UDP

Should we choose speed or reliability?

Two Different Conversations

TCP

Reliable communication.

  • Banking
  • Emails
  • HTTPS
  • Software Updates

UDP

Fast communication.

  • Gaming
  • Streaming
  • Video Calls
  • DNS Requests

Real World Example

Imagine transferring:

10,000 SAR

Would you choose speed?

or

Reliability?

TCP is preferred because losing a packet could corrupt important information.

Video Call Example

Imagine joining an online meeting.

Would you rather:

  • Lose one frame?
  • Or freeze for three seconds?

UDP is usually preferred because speed matters more than perfection.

TCP Three Way Handshake

SYN
"Can we communicate?"
SYN / ACK
"Yes, I received your request."
ACK
"Connection established."
Reliable communication starts here.

Security Perspective

TCP

Reliable and easier to follow during investigations.

UDP

Fast communication that may be abused during attacks such as amplification attacks.

04

Common Protocols

Protocols tell us HOW devices communicate across networks.

HTTP

Port 80

HTTP transfers web traffic without encryption.

Where do we see HTTP?

  • Web Pages
  • Downloads
  • Images

Why Should We Care?

HTTP traffic is NOT encrypted.

Anyone monitoring the traffic may be able to view the contents.

Attackers may abuse insecure communications when sensitive information is transmitted.

HTTPS

Port 443

HTTPS provides encrypted communication between devices.

Real World Example

You are logging into:

Your Bank Account

Would you prefer encrypted or unencrypted communication?

HTTPS protects sensitive data during transmission.

Why Attackers Like HTTPS

Encrypted traffic protects users.

However, attackers may also hide malicious communications inside encrypted traffic.

DNS

Port 53

DNS translates names into IP addresses.

How Does DNS Work?

You type: google.com
DNS receives the request.
An IP address is returned.
Your browser connects to Google.

Why Attackers Like DNS

Attackers may abuse DNS to:
  • Hide communications.
  • Contact malicious domains.
  • Transfer small amounts of data.

SSH

Port 22

SSH provides secure remote access to systems.

System administrators use SSH to manage servers remotely.

SMB

Port 445

SMB enables file and printer sharing across networks.

It is commonly used inside organizations and businesses.

RDP

Port 3389

RDP provides graphical remote desktop access.

Security Perspective

Poorly protected RDP services have frequently been targeted during ransomware incidents.

Organizations should protect remote access services carefully.

Before Security Controls

Firewall Appears

ALLOW ? or BLOCK ?

A firewall examines traffic and decides whether packets should be allowed or denied.

It acts as the first line of defense.

Intrusion Detection System (IDS)

DETECT

IDS monitors network activity and generates alerts when suspicious behaviour appears.

IDS can tell us that something bad is happening.

Intrusion Prevention System (IPS)

DETECT + STOP

IPS detects malicious traffic and automatically blocks it.

IPS does not simply observe.

It takes action.
