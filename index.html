TCP vs UDP

Reliability, speed, and what attackers exploit.

TCP vs UDP comparison

TCPUDP
ConnectionHandshakeNo handshake
Use caseAccuracySpeed
ExampleBanking loginVideo call
Security clueFlags and sessionsVolume and unusual destinations

TCP Handshake

SYN →← SYN/ACKACK →

A repeated SYN without completion can indicate scanning or blocked access.

Banking example: choose TCP

When money moves, every byte matters. TCP retransmits missing pieces and keeps order.

Video call example: choose UDP

When a voice packet is late, replaying it is worse than skipping it. UDP prioritizes flow.

Security angle

TCP scans reveal open doors.
UDP floods can overwhelm services.
Both become evidence when paired with time, host, and intent.

Individual Checkpoint

Which protocol family would you expect for DNS lookups, and why might attackers abuse it?

Often UDP/53; attackers can use frequent or patterned DNS requests for tunneling or beaconing.

04

Common Protocols

Default ports, purpose, usage, and security considerations.

HTTP

Port 80

Transfers web content in cleartext. Used by websites and APIs. Security concern: content and credentials can be observed or modified.

HTTPS

Port 443

HTTP over TLS. Used for secure websites, cloud apps, and APIs. Security concern: encrypted payload hides content, but metadata still matters.

DNS

Port 53 UDP/TCP

Translates names to IPs. Used before most web connections. Security concern: suspicious domains, tunneling, and malicious redirects.

SSH

Port 22

Secure remote command-line administration. Security concern: brute force attempts, exposed servers, stolen keys.

SMB

Port 445

Windows file and printer sharing. Security concern: lateral movement, exposed shares, ransomware propagation.

RDP

Port 3389

Remote desktop access. Security concern: password spraying, exposed remote access, suspicious login geography.

They often run web panels, outdated firmware, and remote management services.

Default credentials

admin/admin is not a security strategy.

Beginner example: a camera joins Wi‑Fi, keeps its factory password, then becomes an attacker’s foothold.

Compromised device behavior

  • Unexpected outbound connections.
  • DNS lookups to strange domains.
  • Traffic spikes at regular intervals.
  • Connections to many external IPs.
06

Network Security Controls

Firewall, IDS, and IPS in action.

Firewall

Decision: allow or block traffic based on rules.

Scenario: block inbound RDP from the internet; allow HTTPS outbound to business apps.

IDS

Decision: alert when traffic looks suspicious.

Scenario: detect a port scan and create an alert for the analyst.

IPS

Decision: actively stop suspicious traffic.

Scenario: drop exploit traffic before it reaches a vulnerable server.

Controls comparison

ControlPrimary actionRisk
FirewallPermit/deny by policyBad rules create gaps
IDSAlertNoise and missed alerts
IPSBlockFalse positives can disrupt work
